## 1. 背景

### 1.1 项目缘起

2026 年 5 月 19 日，在福建屏南龙潭村的"乡建黑客松"预热期间，作者（黄喆）与当地新村民源哥、小梅庄主、警务白宇及一批数字游民进行了多轮深度访谈。访谈记录在 `ref/主理人分身/` 目录下（多段录音转写，超过 5 万字）。

访谈中，所有人收敛到一个共同方向：**用 AI 数字分身解决龙潭村乡村振兴示范点中"主理人 IP 难规模化、访客难精准匹配"的痛点**。

本项目作为黑客松（5 月 21-24 日，48 小时极限共创）**软件赛道**的参赛作品，目标是在 48 小时内交付一个可演示、可扫码体验的 Web Demo，并具备后续在龙潭村及其他乡村振兴示范点落地复制的潜力。

### 1.2 龙潭村现状

- 福建屏南龙潭村是**全国乡村振兴示范点**，被纳入"学习强国"宣传案例
- 由林正禄等老师 2015 年起逐步引入艺术家、文化人作为"新村民"，形成数十个个性化空间（民宿、工作室、咖啡馆、艺术空间）
- 现有承载 IP 的小程序（DAO 龙潭、文创熙岭）以**空间陈列**为主，**人与人的互动几乎不可见**
- 携程等流量平台的进入正在让龙潭村滑向"商业化、同质化、低价内卷"——这是中国所有"半红半火"乡村的标准死亡路径（北京新庄、陕西袁家村均已经历）

## 2. 痛点

访谈中明确浮现的真实痛点，按需方分类。

### 2.1 主理人侧（在地空间运营者）

| 痛点 | 真实出处 |
|------|---------|
| **重复讲故事消耗精力** | 警务白宇住进 138 数字游民基地，主理人浅予每天给来访者重复讲解前世今生，"讲个一年试试，吐都吐出来" |
| **接待错配，质量被平台稀释** | 小梅庄主：携程逼着客人去 evaluating 五星级标准、按人头加价被投诉、对自然乡村心生反感的客人比比皆是 |
| **个人 IP 无法在线上立体呈现** | 小梅庄主有视频号、朋友圈、几年前的公众号，但全是零散的、没人整理过的素材；客人到达前对她"为什么从律师转身建小梅庄"几乎一无所知 |
| **主理人没有手段筛选同频客人** | "我的时间和精力也挺宝贵的，不想消耗在不合适的人身上"——小梅庄主 |

### 2.2 访客侧（数字游民、城市向往乡村者）

| 痛点 | 真实出处 |
|------|---------|
| **携程式平台搜不到真实文化氛围** | 黄喆（作者）从北京来龙潭，用 AI 助理订房只能在携程上比价，搜不到 138 这种数字游民基地 |
| **无法在到达前判断空间是否同频** | 同样去乡村，有人要五星级标准、有人要纯自然，错配后双方都难受 |
| **想成为"云村民"但缺乏入口** | "等孩子上大学了我就来"，但前期没有任何方式跟村庄建立线上连接 |

### 2.3 政府/治理侧

| 痛点 | 真实出处 |
|------|---------|
| **乡村振兴示范点缺乏复制范式** | 央视党委书记黄老师评价该方案"开创性强，可复制、可推广，契合城乡融合政策导向" |
| **数字游民价值难以可视化** | 数字游民对乡村的贡献目前停留在媒体报道层面，缺乏产品化承载 |

## 3. 解决方案

### 3.1 一句话

**项目名：「元家乡 2050」**

"元"取意道教元始（本源、混沌、阴阳未分）+ 智能乡村迈向 2050 的未来感。

**项目定义**：把龙潭村主理人（含在地实体主理人与纯线上云村民）的访谈、公众号、朋友圈数据蒸馏成 AI 数字分身。访客在 Web 上先跟"村管家"深入对话，村管家挖掘其个性化需求后生成定制化旅居方案（含主理人 IP、空间、独特产品和服务的组合），再与具体主理人分身深聊，最终在线下单。地图层同时呈现**实空间**（在地物理空间）与**云空间**（云村民虚拟空间）两类节点，体现"虚实结合"的乡村数字化理念。

### 3.1.0 哲学底色：虚实结合 / 阴阳两轨

源自访谈中源哥提出的核心创意：本项目把中国传统"阴阳虚实"哲学落地到产品形态。

- **阴 / 实 / 在地空间**：龙潭村真实存在的民宿、工作室、咖啡空间；主理人是已经搬到村里居住的"新村民"
- **阳 / 虚 / 云空间**：城市数字游民、远程艺术家、设计师在云端的虚拟空间；他们尚未线下入驻，但已通过云空间提供 IP / 服务 / 作品

两轨在同一张地图上并存、互动、互相滋养：
- 实空间主理人的精力被 AI 释放，腾出时间深度接待真正同频的访客
- 云空间村民先在线上验证 IP，攒够互动后再考虑落地为实空间
- 访客通过同一个村管家入口，既可探索实空间体验，也可探索云空间合作

这套"虚实双轨"是本项目对"未来乡村应该长什么样"的回答：**不只是物理空间的数字化，更是把云上的人才和创意纳入村庄生态**。

### 3.1.1 项目核心立场：AI 释放人，而非替代人

本项目的 AI 应用方向与城市企业 AI 形成鲜明对比：

- **城市企业 AI**：典型应用是"蒸馏掉同事/员工"——把员工的能力沉淀到 AI 后裁员，AI 是替代人的工具
- **本项目 AI**：把主理人从"重复事务性工作"中释放出来，让他们把宝贵的精力投入到真正有创造性的工作和生活中

具体场景对比：

| 场景 | 没有 AI 之前 | 有了数字分身 |
|------|-------------|------------|
| 接待新客人介绍空间 | 主理人每天对不同访客重复讲同一段故事，"讲个一年试试，吐都吐出来" | 数字分身代劳，主理人留出大块时间画画、写诗、做手作、休息 |
| 接待意向客人前的需求摸底 | 加微信→反复问答→可能错配后失望→双方耗能 | 访客先和分身聊够，到线下时已经是高度同频 |
| 个性化推荐与方案设计 | 主理人个人能力受限于"一店一品" | 村管家能跨空间整合，组合出"住小梅庄 + 跟刚子学画 + 在吴桐处听音乐疗愈"的定制方案 |
| 主理人创造性时间 | 被运营、接待、上传携程图片、改价等琐事填满 | AI 接管这些重复劳动，主理人可以真正回到艺术、生活、思考本身 |

**这是我们对"AI 时代乡村应该长什么样"的回答：AI 不应该让乡村的人更卷，而应该让乡村的人更松。**

### 3.2 核心价值主张

| 角色 | 价值 |
|------|------|
| 主理人 | **被释放**：不再重复讲故事，AI 分身代劳 100 遍接待对话；自动过滤错配客人；从运营琐事中解脱出来，把精力还给创作、生活、深度思考 |
| 访客 | **被深度服务**：村管家主动深挖个性需求，给出定制化旅居方案（不只是推荐空间，而是组合主理人 IP + 空间 + 独特产品和服务）；到达前已跟分身建立同频，落地零摩擦 |
| 数字游民 / 云村民 | **本期 demo 已覆盖**：把自己也"蒸馏上去"成为云村民，无需肉身在场，先在云空间立 IP / 提供服务 / 与在地主理人形成跨地域协作。作者本人（黄喆）即作为示例云村民出现在 demo 地图中 |
| 政府 | 可复制的乡村振兴数字化范式，契合"城乡融合""学习强国"政策；输出乡村文化 IP 的标准化路径 |
| 评委 | 看到一个真实可信、可解释、可审计的 AI 应用，每条 AI 输出都能追溯到原文；且立场鲜明 —— 用 AI 让乡村的人更松，不是更卷 |

### 3.3 产品形态（黑客松 Demo 范围）

**Web 应用**，扫码即用，无需注册。

**三层界面**：

```
龙潭村地图层（村管家 agent 驻场，提供"个性化定制旅居方案"）
   │  实空间 (实心点位 / 黑) + 云空间 (虚化点位 / 白) 并列呈现
   │  视觉上参照太极阴阳虚化, 体现虚实结合理念
   │
   │  对话挖掘需求 → 输出方案（地图高亮 + 推荐理由 + 行程/IP/服务组合）
   ↓
空间内页（活动 + 作品为主，主理人分身对话，商品/服务次优）
   │  实空间内页: 物理空间介绍 + 在地主理人分身
   │  云空间内页: 虚拟空间介绍 + 云村民分身 + 提供的远程服务
   │
   │  访客可在不同空间间往返切换、继续与方案中其他主理人深聊
   │  每次有效互动获得"游戏积分" (访问空间 / 对话主理人 / 浏览活动作品)
   ↓
模拟下单（mock 订单确认，演示完整商业闭环）
```

**实空间 vs 云空间的视觉区分**：

| 维度 | 实空间 | 云空间 |
|------|-------|-------|
| 地图点位样式 | 实心圆 / 黑色 / 清晰边缘 | 半透明 / 白色或银色 / 虚化光晕 |
| 主理人来源 | 已在龙潭村定居的新村民 | 城市远程数字游民 / 远程艺术家 |
| 提供的产品/服务 | 住宿、线下活动、空间体验 | 远程咨询、AI 服务、远程课程、作品定制 |
| 进入空间后的内页 | 含真实空间照片 + 活动 + 作品 | 含虚拟空间示意 + 提供的远程服务 + 作品 |

### 3.3.1 游戏化积分系统（简略实现）

为了体现"虚实结合的游戏化乡村"理念，访客在 demo 中通过有效互动累积**元家乡积分（YuanXP）**：

| 行为 | 获得积分 |
|------|---------|
| 与村管家完成一轮深度需求对话 | +10 |
| 进入某个空间内页 | +5 |
| 与主理人分身进行 5 轮以上对话 | +20 |
| 浏览空间的活动或作品 | +5 |
| 完成 mock 下单 | +30 |

积分实时显示在页面右上角小部件，扫码会话期间持续累积，存 localStorage。

**积分的演示意义**（非实际功能）：
- 评委演示中提一句"积分可在 Phase 2 兑换云空间建设权 / 兑换主理人优惠 / 兑换龙潭村文创产品"
- 体现"游戏化乡村社区"的未来感，不实现兑换逻辑
- 复杂度高的话可作为优化清单延后

**实现策略**：极简——前端一个 Zustand store 或 React Context 持有 `xp` 数字，关键 action 触发 `+xp` 动画。不接后端、不持久化到 vault、不影响 LLM prompt。预计实现成本 1-1.5h。

**村管家能力升级说明**：

村管家"龙寻"是一个在龙潭村住了 10 年的"虚构数字村民"。它有 5 项能力：

1. **讲村史**：基于《龙潭造梦记》等史料，可以叙述龙潭村的发展历程、关键人物、历史事件、非遗项目、景点。每条史实可追溯到原书页码。
2. **挖需求**：主动追问访客的旅居目的、性格偏好、时间预算、特殊关注（带娃 / 创作 / 康养 / 艺术 / 安静）、对乡村生活的具体期待
3. **画画像**：归纳出访客的"旅居人群类型"（亲子 / 数字游民 / 退休康养 / 艺术家 / 普通体验者）
4. **出方案**：组合推荐主理人 IP（人）+ 空间（住宿/工作场所）+ 独特产品/服务（活动、课程、作品、本地特产），形成一份可执行的定制化旅居方案
5. **路由空间**：当访客已选定方向，简短介绍空间和主理人特色，引导深入对话由该空间的主理人分身接管

**3 个主理人分身**（具体人选待定）：
- 2 个**深度蒸馏**主理人（小梅庄主已确定 + 第 2 个待选）—— 完整人设、对话、活动、作品、evidence_ledger
- 1 个**占位主理人** —— 仅地图上可见，进入空间内页显示"分身即将上线"。这是 Phase 2 扩展位的展示，向评委传达"系统可复制"的信号

### 3.4 核心差异化

相比同期可能出现的其他 AI 应用赛道方案，本项目的差异化在于：

1. **立场鲜明的 AI 应用价值观**——AI 释放主理人精力做创造性工作，不是替代人。这是与主流"蒸馏员工后裁员"方向截然相反的乡村 AI 范式
2. **村管家不是搜索引擎，是定制方案设计师**——会主动挖需求、画画像、组合 IP + 空间 + 产品，输出"定制旅居方案"而非空间列表
3. **每条 AI 回答可追溯到 markdown 源文件**——不是黑盒训练，是结构化蒸馏 + 引用透明
4. **wiki 作为人类可读知识层**——主理人本人、政府均可审查与修订（dot-skill 的 Correction Log 机制）
5. **运行时显式 abstain 机制**——分身不会编造主理人没说过的话（PersonaCite 论文方法）
6. **真实落地的乡村文化场景**——非工作/职场分身，非娱乐 roleplay，乡村 lifestyle IP 在所有蒸馏项目里是空白
7. **48 小时单人 + AI vibe coding 全程交付**——本身就是 AI 原生开发的范式案例

- 同时争取以下任一特别奖：最具落地实用性 / 最具创意 / 最佳技术

- 演示后至少 1 位主理人愿意签字授权"用我的数据继续做这个项目"
- 演示后至少 1 位政府/媒体方表达"想了解能否在其他村复制"的意向

**技术维度**：
- 全部 AI 回答可在 30 秒内回溯到 vault 中的源文件
- LLM provider 可在 5 分钟内现场切换（.env 重启）
